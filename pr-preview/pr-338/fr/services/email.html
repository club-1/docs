Messagerie email#

Chaque membre dispose d’une boîte email avec une adresse email personnelle. L’adresse est composée de l’identifiant, suivi de @club1.fr. Par exemple, l’adresse de l’utilisateur michel est michel@club1.fr.

Pour utiliser votre boîte mail, il faut vous y connecter à travers un client mail.

Si vous utilisez déjà un client email installé sur votre ordinateur (Thunderbird, Apple mail…) pour gérer vos autres boîtes email, vous pouvez y ajouter celle de CLUB1. Pour cela, vous aurez probablement besoin des informations de connexion.

Sinon, vous pouvez aussi vous connecter depuis un naviguateur Web grâce au client Web.

Enfin, pour les membres utilisant régulièrement la connexion SSH, il est possible de consulter ses mails avec mutt.

Client Web#

Un client Web fourni par Roundcube est disponible à l’adresse https://mail.club1.fr.

Il permet de gérer et envoyer des emails ainsi que de configurer des filtres pour trier automatiquement les emails reçus.

Mais il permet aussi d’accéder aux calendriers et carnets d’adresses CLUB1 synchronisés avec WebDAV.

Stockage des données#

Toutes les données relatives aux emails sont stockées sous la forme de fichiers dans le dossier mail de votre espace personnel. Ce dossier respecte le standard Maildir, il est donc possible d’y accéder directement à partir d’un client email supportant ce standard, par exemple après avoir fait une sauvegarde du dossier sur un ordinateur personnel.

Quotas#

Le dossier mail est soumis à un quota de 5 Go. Cette décision a été prise pour encourager la suppression des anciens emails.

La taille maximale des emails (reçus et envoyés) est de 25 Mo. Si un email dépasse cette taille, il sera refusé et l’expéditeur en sera informé avec le message d’erreur : Message size exceeds fixed limit ou 552: 5.3.4 Error: message file too big.

Dossiers spéciaux#

Les dossiers spéciaux utilisent les noms standards, ils seront donc automatiquement reconnus par la grande majorité des clients email. Au cas où ils ne le sont pas, voici la liste complète :

  • INBOX : messages reçus

  • Drafts : brouillons en cours de rédaction

  • Sent : messages envoyés

  • Junk : SPAMs (automatiquement supprimés au bout de 60 jours)

  • Trash : corbeille (automatiquement supprimés au bout de 60 jours)

  • Archive : messages archivés

Sous-adresses#

En plus de l’adresse principale, les membres ont la possibilité d’utiliser les sous-adresses. Les sous-adresses sont comme des alias. Par défaut, elles redirigent vers l’adresse principale. Toutes les adresses de la forme membre+[...]@club1.fr sont des sous-adresses de membre@club1.fr

Par exemple, tous les emails envoyés à michel+travail@club1.fr seront redirigés vers michel@club1.fr.

Ces adresses peuvent paraitre inutiles à première vue. Mais combinées avec les transferts automatiques et les filtres automatiques, elles deviennent très intéressantes.

Important

Pour le transfert automatique d’une sous-adresse en particulier, on utilise un fichier .formward+[...] au lieu du fichier .forward de l’adresse principale.

Par exemple pour rediriger uniquement l’adresse michel+travail@club1.fr il faut le faire dans le fichier .forward+travail.

Transferts automatiques#

Il est possible de transférer automatiquement les emails que vous recevez sur votre adresse CLUB1 vers une ou plusieurs autres adresses de votre choix. Pour ce faire, il faut créer dans votre espace personnel un fichier qui va indiquer à Postfix à qui transférer votre courrier entrant. Ce fichier doit être placé directement dans votre dossier et s’appeler .forward.

Avertissement

.forward est un fichier caché.

À l’intérieur de ce fichier, il suffit ensuite de lister toutes les adresses vers lesquelles vous souhaitez rediriger les emails en les séparant par un retour à la ligne.

Exemple de fichier .forward transférant vers deux adresses emails :

perso@serveur.net
unautremail@cool.fr

Astuce

Il est même possible de conserver ses emails sur sa boîte CLUB1 en ajoutant son nom d’utilisateur·trice à cette liste !

Pour mettre fin à une redirection, il suffit de supprimer la ligne correspondante. Il est aussi possible de supprimer le fichier pour tout annuler.

Filtres automatiques#

Les filtres Sieve permettent de trier automatiquement les emails que vous recevez. Ces filtres sont appliqués côté serveur et ne dépendent donc d’aucun client. Chaque utilisateur doit cependant les configurer lui-même.

Il existe deux manières de les éditer. La plus simple est d’utiliser le client Web Roundcube de CLUB1. Il dispose dans ses paramètres d’une interface de gestion de filtres.

La deuxième méthode consiste à directement modifier le fichier de filtre dans votre espace personnel :

~/.dovecot.sieve

Logiciels#

Postfix#

Serveur SMTP utilisé pour la transmission d’email. — Wikipedia, Sources

Dovecot#

Serveur IMAP utilisé pour la gestion de boîtes emails. — Wikipedia, Sources

Pigeonhole#

Extension Dovecot pour Sieve et ManageSieve, permettant de configurer des filtres automatiques côté serveur. Ces filtres peuvent être édités depuis Roundcube. — Site Web, Sources

SPF-Engine#

Serveur de traitement SPF (Sender Policy Framework) utilisé par Postfix pour vérifier l’origine des emails entrants. — Sources

PostSRSd#

Serveur de traitement SRS (Sender Rewriting Scheme) permettant à Postfix de transférer des emails en restant compatible avec SPF. — Sources

OpenDKIM#

Serveur fournissant un filtre milter utilisé par Postfix pour signer les emails sortants et valider la signature des emails entrants. — Site Web, Sources

OpenDMARC#

Serveur fournissant un filtre milter utilisé par Postfix pour valider la politique DMARC (Domain-based Message Authentication, Reporting, and Conformance) des emails entrants.

Il se base pour cela sur le retour de SPF-Engine et OpenDKIM. — Site Web, Sources

dmarcator#

Serveur fournissant un filtre milter utilisé par Postfix pour rejeter certains emails en fonction du retour d’OpenDMARC. — Sources

Roundcube#

Client Web permettant de consulter et d’envoyer des emails via SMTP et IMAP. — Wikipedia, Sources

Voir aussi

La section Client Web

Mutt#

Client email en CLI permettant d’envoyer et de consulter ses email directement depuis un terminal. Il est installé et pré-configuré sur le serveur CLUB1. — Wikipedia, Sources
